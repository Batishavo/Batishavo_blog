Configurar Clave SSH Git

Paso1

Abre una terminal y verifica si ya tienes una clave SSH en tu sistema ejecutando el siguiente comando: 


                    ls ~/.ssh

Paso 2

Si el directorio ~/.ssh está vacío o no contiene una clave con el formato adecuado, puedes generar una nueva clave SSH. Ejecuta el siguiente comando y reemplaza tu_correo@example.com con tu dirección de correo electrónico asociada a tu cuenta de Git: 


                    ssh-keygen -t rsa -b 4096 -C "tu_correo@example.com"

Paso 3

El agente SSH te ayudará a gestionar tus claves y evitará que tengas que ingresar la contraseña cada vez que te conectes. Ejecuta el siguiente comando para iniciar el agente SSH: 


                    eval "$(ssh-agent -s)"

Paso 4

Añade la clave SSH que generaste al agente ejecutando: 


                    ssh-add ~/.ssh/id_rsa

Paso 5

Para utilizar la clave SSH en tu cuenta de Git, necesitas copiar la clave pública al portapapeles. Puedes hacerlo con el siguiente comando: 


                    cat ~/.ssh/id_rsa.pub

Paso 6

Accede a la plataforma web de tu proveedor de Git (GitHub, GitLab, Bitbucket, etc.), inicia sesión en tu cuenta y navega a la sección de configuración de claves SSH. Pega la clave pública que copiaste en el paso anterior.

Paso 7

Para asegurarte de que todo esté configurado correctamente, intenta clonar un repositorio utilizando SSH en lugar de HTTPS. Por ejemplo: 


                    git clone git@github.com:usuario/nombre-repositorio.git

Con estos pasos, habrás configurado tu clave SSH en Ubuntu para utilizarla con Git. Ahora podrás realizar operaciones en tus repositorios de Git sin tener que ingresar tu contraseña cada vez que interactúes con ellos.
